╔═══════════════════════════════════════════════════════════════════════════╗
║ ROUTING DECISION TREE - WHY PROACTIVE DOESN'T REACH DISCORD/TELEGRAM    ║
╚═══════════════════════════════════════════════════════════════════════════╝

    shouldRouteToOriginating = 
        isRoutableChannel(originatingChannel) AND
        originatingTo AND
        originatingChannel != currentSurface

    For PROACTIVE MESSAGE:
    ─────────────────────
    originatingChannel = "webchat"
              ↓
    isRoutableChannel("webchat")?
              ↓
    if (!channel || channel === INTERNAL_MESSAGE_CHANNEL) {
      return false;  ← ← ← STOPS HERE, returns FALSE
    }
              ↓
    Result: FALSE AND undefined AND ... = FALSE
              ↓
    shouldRouteToOriginating = FALSE
              ↓
    MESSAGE STAYS INTERNAL ✗ No Discord, no Telegram, no email

    For USER MESSAGE FROM TELEGRAM:
    ──────────────────────────────
    originatingChannel = "telegram"
    originatingTo = chat_id
    currentSurface = "telegram"
              ↓
    isRoutableChannel("telegram")?
              ↓
    if (!channel || channel === INTERNAL_MESSAGE_CHANNEL) {
      // SKIPPED - "telegram" is routable
    }
    return true;  ← ← ← RETURNS TRUE
              ↓
    Result: TRUE AND chat_id AND (telegram == telegram is FALSE)
              ↓
    shouldRouteToOriginating = FALSE (same channel)
              ↓
    But if message needs to go to Discord:
    currentSurface = "discord"
              ↓
    Result: TRUE AND chat_id AND (telegram != discord) = TRUE
              ↓
    shouldRouteToOriginating = TRUE
              ↓
    MESSAGE ROUTES TO DISCORD ✓
