(internal channel) which CANNOT route to external channels.

ARCHITECTURE OVERVIEW
================================================================================

TRIGGER MECHANISM:
  • Scheduler fires at 5 different times daily:
    - Morning:    5:30 - 7:30 (random)
    - Lunch:      11:30 - 13:00 (random)
    - Evening:    20:00 - 22:30 (random)
    - Late-night: 23:00 - 00:30 (random)
    - Nudges:     2.5 - 5 hours apart (during 6 AM - 10 PM)

MESSAGE FLOW:
  1. Scheduler fires    → calls onTrigger()
  2. Invokes chat.send  → with sessionKey="proactive"
  3. Creates context    → OriginatingChannel="webchat" (INTERNAL)
  4. Dispatch           → Check shouldRouteToOriginating
  5. NOT routable       → Process locally instead
  6. Agent replies      → Stream deltas (50ms throttle)
  7. Emit final         → Broadcast to WebSocket + enqueue iOS push
  8. iOS polling        → GET /arona/push/pending or /long-poll
  9. Display            → User sees notification on iOS app

DELIVERY CHANNELS:
  ✓ iOS app            (push notification via long-poll or BGAppRefresh)
  ✓ WebSocket clients  (live broadcast if connected)
  ✓ Session transcript (saved to "proactive" session)
  ✗ Discord            (internal channel, cannot route)
  ✗ Telegram           (internal channel, cannot route)
  ✗ Email              (not supported)

CRITICAL DECISION POINT
================================================================================

The moment that prevents routing to external channels:

FILE: src/auto-reply/reply/dispatch-from-config.ts
LINE: ~200 (in dispatchReplyFromConfig function)

CODE:
  const shouldRouteToOriginating = Boolean(
    isRoutableChannel(originatingChannel) &&    // ← "webchat" → FALSE
    originatingTo &&                            // ← undefined → FALSE
    originatingChannel !== currentSurface
  );

RESULT:
  For proactive: shouldRouteToOriginating = FALSE
  → Message is NOT routed to external channels
  → Message is processed locally instead

DEFINITION OF isRoutableChannel:
  export function isRoutableChannel(
    channel: OriginatingChannelType | undefined,
  ): channel is Exclude<OriginatingChannelType, typeof INTERNAL_MESSAGE_CHANNEL> {
    if (!channel || channel === INTERNAL_MESSAGE_CHANNEL) {
      return false;  // ← "webchat" returns FALSE here
    }
    // ... checks for telegram, discord, slack, signal, etc.
  }

WHAT THIS MEANS:
  • "webchat" is INTERNAL_MESSAGE_CHANNEL
  • Internal channels CANNOT route to external destinations
  • Proactive messages are INTENTIONALLY kept internal
  • This is a design decision, not a bug

================================================================================
STREAMING MECHANICS
================================================================================

DELTA THROTTLING:
  • emitChatDelta() is called for each token from the agent
  • Deltas are only sent to clients every 50ms MINIMUM
  • If a delta has been sent within 50ms, it's skipped
  • This prevents overwhelming WebSocket clients

BROADCAST BEHAVIOR:
  • broadcast("chat", payload, { dropIfSlow: true })
    - Message dropped if WebSocket buffer fills
    - Ensures gateway doesn't block on slow clients

SEQUENCE TRACKING:
  • Each message has a seq number
  • Clients can detect gaps in sequence
  • Helps detect message loss or reordering

BUFFERING:
  • Text is buffered in chatRunState.buffers
  • Used to accumulate multi-part responses
  • Cleaned up after final message sent

ENQUEUE:

CONCLUSION
================================================================================

The Arona proactive system is working as designed:

✓ Scheduler fires at randomized times within configured windows
✓ Messages are sent to the internal "proactive" session
✓ Text is streamed to WebSocket clients (if any connected)
✓ iOS push notifications are always enqueued
✓ Messages are saved to session transcript

× Proactive messages CANNOT route to Discord/Telegram/email
× This is INTENTIONAL - proactive is for personal notifications only
× The system explicitly blocks external routing at shouldRouteToOriginating

The system is designed for local, personal AI companion notifications,
not for broadcasting to team channels or external systems.

For messages to reach external channels (Discord, Telegram, etc.),
they must:
1. Be triggered by a user message on that channel, OR
2. Be explicitly sent via an external API/tool

Proactive messages are internal-only by architectural design.
