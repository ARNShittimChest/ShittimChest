================================================================================
                          NETWORKING ARCHITECTURE
================================================================================

┌─────────────────────────────────────────────────────────────────────────────┐
│                      WEBSOCKET CONNECTION FLOW                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  STEP 1: DISCOVERY                                                          │
│  ┌──────────────────────────────────────────────────────────────────────┐  │
│  │ GatewayDiscoveryModel (Bonjour/mDNS)                                │  │
│  │  - Scans local network for _shittimchest-gw._tcp                   │  │
│  │  - Returns: [ DiscoveredGateway { name, stableID, host, port } ]   │  │
│  └──────────────────────────────────────────────────────────────────────┘  │
│                               ↓                                             │
│  STEP 2: TLS FINGERPRINT VERIFICATION                                      │
│  ┌──────────────────────────────────────────────────────────────────────┐  │
│  │ First connection: User manually verifies fingerprint                │  │
│  │  - TCPProbe.swift reads TLS certificate from gateway               │  │
│  │  - Display SHA256 fingerprint to user                              │  │
│  │  - User confirms "Yes, I trust this gateway"                       │  │
│  │  - Store fingerprint in GatewayTLSStore (Keychain)                 │  │
│  │                                                                      │  │
│  │ Subsequent connections:                                             │  │
│  │  - Load stored fingerprint from GatewayTLSStore                    │  │
│  │  - Verify incoming certificate matches                             │  │
│  │  - Refuse connection if fingerprint mismatch (TOFU+pinning)        │  │
│  └──────────────────────────────────────────────────────────────────────┘  │
│                               ↓                                             │
│  STEP 3: WEBSOCKET CONNECT                                                 │
│  ┌──────────────────────────────────────────────────────────────────────┐  │
│  │ GatewayNodeSession.connect()                                        │  │
│  │  - URL: wss://[host]:[port]/  (TLS required)                       │  │
│  │  - Auth headers: X-Auth-Token, X-Auth-Password                     │  │
│  │  - Payload: DeviceIdentity { id, instanceId, nodeType }            │  │
│  │                                                                      │  │
│  │ TWO CONCURRENT SESSIONS:                                            │  │
│  │  1. nodeGateway     - Device capabilities (camera, location, etc)  │  │
│  │  2. operatorGateway - Chat, Talk, Configuration                    │  │
│  └──────────────────────────────────────────────────────────────────────┘  │
│                               ↓                                             │
│  STEP 4: RPC MESSAGE FLOW                                                  │
│  ┌──────────────────────────────────────────────────────────────────────┐  │
│  │                                                                      │  │
│  │  Method Call Pattern (node.invoke):                                 │  │
│  │  ┌─────────────────┐                                                │  │
│  │  │ Client          │                                                │  │
│  │  │ (iOS app)       │                                                │  │
│  │  └────────┬────────┘                                                │  │
│  │           │ await send(method: "node.invoke")                       │  │
│  │           │ params: {id, instruction, params}                       │  │
│  │           ↓                                                          │  │
│  │  ┌────────────────────┐                                             │  │
│  │  │ WebSocket Channel  │                                             │  │
│  │  │ (GatewayChannel)   │                                             │  │
│  │  └────────┬───────────┘                                             │  │
│  │           │ TLS, encrypted, JSONL                                   │  │
│  │           ↓                                                          │  │
│  │  ┌────────────────────┐                                             │  │
│  │  │ ShittimChest       │                                             │  │
│  │  │ Gateway            │                                             │  │
│  │  │ (Server)           │                                             │  │
│  │  └────────┬───────────┘                                             │  │
│  │           │ Process instruction (e.g., camera.snap)                 │  │
│  │           │ Return result                                            │  │
│  │           ↓                                                          │  │
│  │  ┌────────────────────┐                                             │  │
│  │  │ Callback Handler   │                                             │  │
│  │  │ handleInvokeResp() │                                             │  │
│  │  └────────────────────┘                                             │  │
│  │                                                                      │  │
│  │  Event Streaming Pattern (node.event):                              │  │
│  │  - VoiceTranscript: voice wake detected                             │  │
│  │  - LocationUpdate: significant location change                      │  │
│  │  - Custom: whatever the gateway needs to know                       │  │
│  │                                                                      │  │
│  └──────────────────────────────────────────────────────────────────────┘  │
│                               ↓                                             │
│  STEP 5: PUSH NOTIFICATIONS (APNs)                                         │
│  ┌──────────────────────────────────────────────────────────────────────┐  │
│  │ At launch: registerForRemoteNotifications()                         │  │
│  │  → iOS gives APNs device token                                      │  │
│  │  → Send to gateway via node.invoke("push.apns.register")            │  │
│  │                                                                      │  │
│  │ Gateway sends silent push (background wake):                        │  │
│  │  → didReceiveRemoteNotification() callback                          │  │
│  │  → appModel.handleSilentPushWake(userInfo)                          │  │
│  │  → Reconnect if needed, sync state, post watch notifications        │  │
│  │                                                                      │  │
│  │ Gateway sends user notification (watch prompt mirroring):           │  │
│  │  → userNotificationCenter(didReceive:) callback                     │  │
│  │  → Parse watch prompt metadata from userInfo                        │  │
│  │  → User taps action → handleMirroredWatchPromptAction()             │  │
│  │                                                                      │  │
│  │ Background app refresh (BGTaskScheduler):                           │
│  │  → System triggers every 15+ minutes                                │  │
│  │  → appModel.handleBackgroundRefreshWake()                           │  │
│  │                                                                      │  │
│  └──────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

================================================================================
                    INITIALIZATION & APP LIFECYCLE
================================================================================

LAUNCH SEQUENCE
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │
│ 1. main @ShittimChestApp                                                    │
│    └─ Install uncaught exception handler                                    │
│    └─ Initialize NodeAppModel (injects all services)                        │
│    └─ Initialize GatewayConnectionController (starts Bonjour)               │
│                                                                              │
│ 2. didFinishLaunchingWithOptions                                            │
│    └─ Register background refresh task (BGTaskScheduler)                    │
│    └─ Set UNUserNotificationCenter delegate                                 │
│    └─ Call registerForRemoteNotifications() → APNs token request            │
│                                                                              │
│ 3. application(didRegisterForRemoteNotificationsWithDeviceToken:)           │
│    └─ appModel.updateAPNsDeviceToken(token)                                │
│    └─ Send to gateway via node.invoke("push.apns.register")                │
│                                                                              │
│ 4. RootCanvas appears                                                       │
│    ├─ Evaluate onboarding presentation route                               │
│    │  └─ If first-time: show OnboardingWizardView (full-screen)            │
│    │  └─ If no gateway config: show SettingsTab (sheet)                     │
│    │  └─ Otherwise: show canvas                                             │
│    │                                                                        │
│    ├─ Prevent idle sleep if "screen.preventSleep" is true                   │
│    │  └─ UIApplication.shared.isIdleTimerDisabled = true                    │
│    │                                                                        │
│    └─ Update canvas debug status if enabled                                │
│                                                                              │
│ 5. Scene phase changes                                                      │
│    ├─ .active   → Resume discovery, attempt reconnect                       │
│    ├─ .inactive → Pause discovery                                           │
│    └─ .background → Schedule background refresh task, suppress reconnect    │
│                                                                              │
│ 6. User navigates or closes app                                             │
│    └─ Graceful disconnection (but websocket can reconnect on foreground)    │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
